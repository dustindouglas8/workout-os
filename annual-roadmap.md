# Annual Roadmap System

## Purpose

Workout OS plans the full year directionally while only detailing the current block and the next block. This keeps the program pointed at the user's long-term health and strength goals without pretending that month 9 can be programmed accurately before month 3 results exist.

## Time Horizons

| Layer | Duration | Detail Level | Purpose |
|---|---:|---|---|
| Annual roadmap | 12 months | Directional | Set the sequence of major goals. |
| Block | 8-12 weeks | Detailed for current + next block only | Drive one primary adaptation. |
| Phase | 3-4 weeks | Fully detailed inside active block | Progress, test, and adjust. |
| Week | 7 days | Fully detailed for current block | Execute sessions and recovery. |
| Day | 1 day | Fully detailed | Follow readiness, practice, training, and logging flow. |

## Default 12-Month Sequence

The default annual plan uses four 12-week blocks:

1. **Recomp + tissue capacity + movement quality**
   - Goal: improve body composition while building connective-tissue tolerance, movement skill, aerobic base, and consistent logging.
   - Best for: most users starting the system, users returning from inconsistent training, and users who need recovery improvements before harder strength work.

2. **Strength / specialization**
   - Goal: shift from general recomp into a measurable strength or skill priority.
   - Examples: pull-up strength, lower-body strength, ring strength, boxing conditioning, mobility specialization.
   - Block 2 should be drafted during Block 1, then finalized after Block 1 tests.

3. **Hypertrophy / performance / endurance / skill emphasis**
   - Goal: build on the adaptation that is lagging or most relevant after Block 2.
   - This block remains directional until Block 2 review.

4. **Consolidation / specialization / repeat cycle**
   - Goal: either consolidate gains, run a new specialization, or repeat recomp/strength based on user outcomes.
   - This block remains directional until Block 3 review.

## Block Types

| Block Type | Primary Adaptation | Use When |
|---|---|---|
| Recomp | Fat loss with performance retention or muscle gain | Body composition is a primary goal and recovery can support training. |
| Strength | Higher force output in selected patterns | Benchmarks show stable recovery and clean movement quality. |
| Hypertrophy | More muscle volume and local work capacity | Strength base is adequate and joint tolerance is stable. |
| Mobility / rehab | Pain reduction, range, tissue tolerance | Pain, injury history, or failed movement screens are limiting progress. |
| Endurance | Zone 2, work capacity, conditioning | Aerobic base limits recovery or sport/skill performance. |
| Skill specialization | Boxing, rings, calisthenics, dance, sport-specific work | User has a defined skill priority and enough recovery budget. |

## Phase Structure Inside A Block

Default 12-week block:

| Phase | Weeks | Intent |
|---|---:|---|
| Phase 1 | 1-3 | Establish technique, baseline volume, and recovery response. |
| Test / deload | 4 | Reduce volume, test benchmarks, review pain/readiness. |
| Phase 2 | 5-7 | Increase density, load, or complexity if gates are passed. |
| Test / deload | 8 | Repeat benchmarks and decide whether to advance or repeat. |
| Phase 3 | 9-11 | Highest specificity for the block goal. |
| Test / transition | 12 | Benchmark, review, and finalize the next block. |

Eight-week and sixteen-week variants use the same idea: 3-4 weeks of loading followed by a test/deload checkpoint.

## Advancement Decisions

Each phase review returns one recommendation:

| Recommendation | Meaning |
|---|---|
| Advance | Benchmarks, pain, readiness, and adherence support the next phase. |
| Repeat | Adaptation is incomplete but the block goal is still appropriate. |
| Deload | Fatigue, pain, or life stress requires a recovery week before deciding. |
| Pivot | The block goal is wrong for the current user state; switch emphasis. |

The system uses gates plus coach override. An override is allowed, but the post-review log must record the reason.

## What Gets Planned When

- At program creation: define the 12-month roadmap, detail Block 1, draft Block 2, and name likely goals for Blocks 3-4.
- At each phase review: update the current block details if the user is under- or over-recovering.
- At Block 1 Week 12: finalize Block 2 from actual results.
- At each block review: revise the remaining annual roadmap.

## Dustin Reference Path

Dustin's likely annual path:

1. Block 1: Recomp + tissue capacity + movement quality.
2. Block 2: Strength/specialization, selected from Block 1 results.
3. Block 3: Hypertrophy, ring strength, boxing conditioning, or mobility emphasis based on bottleneck.
4. Block 4: Consolidation or second specialization.

This path is a reference config, not the universal default.
