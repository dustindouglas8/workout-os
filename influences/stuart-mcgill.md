# Stuart McGill

## Role In Workout OS

Stuart McGill is useful as the spine biomechanics and back-resilience influence. Workout OS should use McGill-style thinking when programming hinges, loaded carries, core work, and pain-modified regressions.

## Core Domains

- Spine biomechanics.
- Back pain assessment and rehabilitation.
- Spine-sparing core endurance.
- Hinge mechanics and load tolerance.
- Matching exercise to the individual's pain triggers.

## Principles To Extract

1. **The back needs capacity, not random punishment**
   - Core work should build stiffness, endurance, and control before fatigue.
   - Workout OS translation: dead bug, side plank, bird dog, carries, and controlled back extensions are programmed before aggressive flexion/extension volume.

2. **Find the pain trigger**
   - Flexion, extension, compression, shear, or rotation may be the issue depending on the user.
   - Workout OS translation: pain notes should include trigger pattern, not just location.

3. **Hinge quality matters**
   - Loaded hinge work requires bracing, hip motion, and neutral spine control.
   - Workout OS translation: RDL, swing, deadlift, and squat records include spine-position cues and regressions.

4. **Core endurance often matters more than max ab strength**
   - A resilient trunk holds position under fatigue.
   - Workout OS translation: use carries, anti-extension, anti-rotation, and endurance holds.

## Workout OS Rules

- Low-back pain modifies pattern selection before intensity.
- Avoid repeated loaded spinal flexion during acute symptoms.
- Use carries and anti-motion core work as default strength-phase trunk work.
- Use stability ball back extension only as low-load capacity work, never as a max effort lift.

## Original Example Sequence

**Spine-sparing core capacity**

```text
1. Dead Bug — 6/side with full exhale
2. Side Plank — 20-30s/side
3. Bird Dog — 6/side with 2s reach
4. Suitcase Carry — 30-40 steps/side if pain-free

Rule:
Stop before form decay. The goal is repeated high-quality bracing, not ab burn.
```

## What Not To Copy

- Do not claim to diagnose back pain.
- Do not copy book protocols wholesale.
- Do not force the same back routine on every user; pain triggers vary.
