# Intake Questionnaire — Canonical Q&A Flow

This is the canonical question sequence for generating a personalized Training OS for any user. These 20 questions were extracted and structured from the ChatGPT conversation used to design Dustin's program. Every question has a defined purpose, answer type, and documented impact on program output.

The goal of this document is to make the intake fully reproducible: anyone running this sequence against any user profile should arrive at a complete program config with no ambiguity.

---

## Q1: Training Frequency Target Per Week
**Category:** Schedule
**Why asked:** Determines the weekly split structure. 6 days requires a different architecture than 3 or 4 — rest distribution, session sequencing, and CNS recovery windows are all affected.
**Answer type:** Single choice
**Options:**
- 3 days/week
- 4 days/week
- 5 days/week
- 6 days/week
**Dustin's answer:** 6 days/week
**Impact on program:** 6 days → choose a split architecture: Lower First, Push First, or Upper Alternating Emphasis. 4 days → compressed split: Lower / Upper / Lower / Upper or Full Body. 3 days → full-body sessions only. Session duration targets also adjust: 6-day programs use 60-min hard caps to prevent cumulative fatigue.

## Q2: When You Travel, What's Typically Available?
**Category:** Equipment
**Why asked:** Establishes the travel floor — the minimum equipment set that must be planned around. Every exercise in the program needs a travel-viable substitute or the plan fails on road weeks.
**Answer type:** Multi-select
**Options:**
- Body weight only (no equipment)
- Resistance bands (own and carry)
- Hotel gym (dumbbells and cables, no barbells)
- Full commercial gym
- Varies / unpredictable
**Dustin's answer:** Resistance bands + hotel gym (dumbbells and cables available most trips)
**Impact on program:** Determines travel substitution library. Band-only floor means every movement needs a band substitute. Hotel gym floor unlocks dumbbell progressions. If "full gym" — travel mode is essentially identical to home mode.

## Q3: Primary Goal for Phase 1 (and Overall Program)
**Category:** Goals
**Why asked:** The primary goal governs calorie targets, volume-to-intensity ratio, phase sequencing emphasis, and what the benchmark tests measure.
**Answer type:** Single choice
**Options:**
- Recomp (simultaneous fat loss + muscle gain)
- Bulk (muscle gain priority, calorie surplus)
- Cut (fat loss priority, calorie deficit)
- Performance (strength or skill peak)
- Maintenance / longevity
**Dustin's answer:** Recomp
**Impact on program:** Recomp → moderate calorie cycling by day type, high protein anchor, training stimulus variety across the week, no extended surplus phases. Bulk → consistent surplus, more compound volume. Cut → deficit with protein preservation, some volume reduction in Phase 3. Performance → periodized intensity peaks, strength benchmarks as primary metric.

## Q4: What Equipment Do You Have at Home?
**Category:** Equipment
**Why asked:** Determines the home-mode exercise library — which movements are available without substitution, which progressions are accessible, and which gaps need creative solutions (hence the vest, sliders, rings).
**Answer type:** Multi-select (checkboxes with optional sub-fields)
**Options:**
- Pull-up bar (type: mounted wall / doorframe / ceiling)
- Gymnastic rings
- Resistance bands (specify: light / medium / heavy / loop)
- Weight vest (specify: fixed weight / adjustable; current load in lbs)
- Sliders / furniture movers
- Kettlebell (specify weight)
- Dumbbell set (specify range)
- Jump rope
- Yoga mat
- Stability ball / exercise ball
- Foam roller
- Lacrosse ball
- Pneumatic compression sleeves (e.g., NormaTec)
**Dustin's answer:** Pull-up bar (mounted), gymnastic rings, resistance bands (light/medium/heavy + loop), weight vest (adjustable), sliders, jump rope, yoga mat, stability ball, pneumatic compression sleeves
**Impact on program:** Full home inventory → vest loading progression available (Phase 2+), sliders open 4 gap-fix movements (Phase 1), rings enable instability layer (Phase 2+), stability ball adds low-back-friendly core flexion, anti-extension, hamstring rehab, and squat-pattern support. No vest → bodyweight must carry load through tempo and unilateral progressions only. No sliders → substitute hamstring curl with Nordic eccentric or single-leg Romanian deadlift.

## Q5: What Day Do You Start the Week?
**Category:** Schedule
**Why asked:** Controls which day gets the highest-CNS session (first training day of the week = most recovered). Also sets the weekly planning rhythm for meal prep and Sunday logging.
**Answer type:** Single choice
**Options:**
- Sunday
- Monday
**Dustin's answer:** Sunday
**Impact on program:** Sunday start → Sunday is the highest-intensity day for the selected split architecture. Weekly baseline log runs Sunday. Meal prep runs Sunday. Monday start → Monday gets the heavy session, Sunday is recovery/mobility.

## Q6: Current Pull-Up Status
**Category:** Constraints / Fitness Baseline
**Why asked:** Pull-up progression is the primary vertical pull movement and one of the 6 benchmark metrics. Current status determines starting phase, volume, and whether assistance is needed.
**Answer type:** Composite (two sub-questions)

Sub-question A: Current max reps (clean form)
- Options: 0 (can't do one) / 1–3 / 4–6 / 7–10 / 10+

Sub-question B: Form quality
- Options: Kipping only / Partial ROM (chin over bar but not chest-to-bar) / Full ROM (dead hang start to chest-approaches-bar) / Weighted

**Dustin's answer:** 4–6 reps, full ROM, dead hang start
**Impact on program:** 4–6 clean reps → start Phase 1 at grease-the-groove approach (sets of 3, sub-maximal, frequent). Phase 2 gate is 3×5 full ROM. Vest introduced only after 3×8 gate. Users at 0 start with ring rows + band-assisted negatives. Users at 10+ start Phase 2 directly.

## Q8: Cardio Type and Current Level
**Category:** Fitness Baseline
**Why asked:** Establishes Zone 2 baseline for Tuesday session programming. Someone running 45 minutes easily needs a different Zone 2 session than someone who walks only.
**Answer type:** Multi-select + free text

Sub-question A: Cardio modes available/preferred (multi-select)
- Walking, Running, Cycling, Rowing, Jump rope, Swimming

Sub-question B: Current aerobic capacity
- Options: Sedentary (< 20 min continuous activity) / Low (walk 30+ min) / Moderate (run/bike 20–30 min) / High (run 45+ min)

**Dustin's answer:** Jump rope (primary), walking as floor; moderate capacity
**Impact on program:** Jump rope → Tuesday session uses jump rope intervals as Zone 2 tool (not running). Low capacity → Zone 2 session starts at 20–25 min, builds. Moderate → 30–35 min. High → 40–45 min with HR cap (< 75% max HR). Zone 2 is always at conversation pace — this is not HIIT.

- Hip flexors / hip mobility

## Q10: Morning Circuit Priority (Activation Design Preference)
**Category:** Preferences
**Why asked:** Determines how the morning circuit is structured — whether it's a fixed daily sequence, entirely day-specific, or a hybrid. This is a design philosophy question, not just a preference.
**Answer type:** Single choice
**Options:**
- Fixed daily (same 4–6 movements every morning regardless of training day)
- Day-specific (different activation movements each morning based on what muscles train today)
- Hybrid (fixed foundation + rotating day-specific additions)
**Dustin's answer:** Hybrid — fixed foundation (F1–F4) + 2 rotating movements by day and phase
**Impact on program:** Hybrid → morning circuit has 4 fixed movements + 2 rotating per day. The fixed 4 are chosen for universal structural needs (rotator cuff, hip flexors, TVA, ankle). The rotating 2 are chosen specifically for what trains today. Fixed-only → simpler, less prep, but no day-specific priming. Day-specific-only → higher cognitive load, harder to automate.

- 12:12 (12 hours eating, 12 fasting)
- 14:10
- 16:8
- 18:6
- 20:4

Sub-question: Window start time and end time

## Q12: Training Time of Day
**Category:** Schedule
**Why asked:** Determines when the session occurs relative to the eating window, and whether fasted training is implied or optional. Also affects warm-up design — morning sessions require more thorough neural preparation than afternoon sessions.
**Answer type:** Single choice
**Options:**
- Early morning (before 8am)
- Morning (8–10am)
- Midday (10am–1pm)
- Afternoon (1–5pm)
- Evening (5–8pm)
- Late night (after 8pm)
**Dustin's answer:** Early morning, fasted (before eating window opens at 8am)
**Impact on program:** Morning fasted → pre-workout amino stack is mandatory (not optional). Warm-up must fully prepare CNS as morning muscle temp is lowest. Session ends at or before 8am to begin eating window. Afternoon training → different warm-up protocol, pre-training feeding may be available, no fasting required.

## Q13: Macro Tracking Level
**Category:** Preferences
**Why asked:** Determines how detailed the nutrition section is and how prescriptive the macro targets are. Strict trackers get exact gram targets. Loose trackers get portion language and food category guidance.
**Answer type:** Single choice
**Options:**
- None (intuitive eating)
- Loose (track protein only, roughly)
- Moderate (track protein + total calories)
- Strict (track all macros: protein, carbs, fat, fiber)
**Dustin's answer:** Moderate (protein + calories; flexible on fat/carb split within target)
**Impact on program:** Moderate → nutrition section shows protein target (1g/lb BW minimum) + daily calorie targets by day type. No gram-level carb/fat tracking required. Strict → all macros calculated and displayed with meal-level breakdown. None → nutrition section shows food category guidance only (e.g., "1–2 palms of protein per meal").

## Q14: First Intake Before Training (Pre-Training Fuel)
**Category:** Preferences / Health
**Why asked:** Determines whether the fasted state is truly fasted or modified-fasted. Some users tolerate nothing before training; others need aminos or caffeine to perform. This affects both the morning sequence and the nutrition section.
**Answer type:** Multi-select
**Options:**
- Nothing (pure fast)
- Water only
- Black coffee or matcha (no additives)
- Amino acids / EAAs
- Creatine
- Full pre-workout stack (caffeine + aminos + other compounds)
- Light food (e.g., banana, handful of nuts)
**Dustin's answer:** Matcha + full pre-workout amino stack (aminos + creatine + caffeine + electrolytes)
**Impact on program:** Amino stack → fasted training is considered "amino-protected" — risk of catabolism reduced. Pre-workout timing is built into morning sequence (after morning circuit, before training). No stack → fasted session is truly fasted; volume may need to be slightly reduced in Phase 1. Full stack → morning sequence explicitly includes stack prep as a step.

## Q15: Sleep Quality (Home vs. Travel)
**Category:** Health / Constraints
**Why asked:** Sleep quality directly affects GYR scoring and the frequency of Yellow/Red days. If travel sleep is chronically poor, travel weeks need built-in volume reduction and the GYR system should weight sleep hours more heavily.
**Answer type:** Two-part (home sleep, travel sleep)

Sub-question: Average hours per night (home):
- Options: < 5 / 5–6 / 6–7 / 7–8 / 8+

Sub-question: Average hours per night (travel):
- Options: Same as home / 1 hour less / 2+ hours less / Highly variable

**Dustin's answer:** Home: 7–8 hours. Travel: 1 hour less than home, variable quality.
**Impact on program:** Home 7–8 → baseline RHR threshold for GYR uses this as normal. Travel 1h less → travel weeks auto-flag as Yellow-likely; travel mode sessions pre-configured for Yellow version. Chronic < 6h home → GYR sleep threshold adjusted down, with explicit note that CNS recovery is compromised; deload more frequently.

## Questions NOT Yet Asked — Gaps for v2 of the Intake

These questions were not asked in the ChatGPT conversation that generated Dustin's program. They would improve personalization or safety for other users and should be added to the v2 intake sequence.

**1. Age and Training Age (years of consistent training)**
- *Why it matters:* Periodization aggressiveness must match training age. A 25-year-old with 3 years of training can handle more volume and higher-intensity deload transitions than a 45-year-old with 1 year. Phase durations and deload frequency should scale with training age.

**2. Full Injury History (not just current rehab targets)**
- *Why it matters:* Past injuries predict future failure points even when fully "healed." A history of shoulder labrum issues changes Phase 3 ring dip programming even if the shoulder feels fine today.

**3. LDL / Cholesterol or Metabolic Health Concerns**
- *Why it matters:* High saturated fat protocols and aggressive recomp cycling can affect lipid panels. If a user has known LDL concerns, the nutrition section should note it and avoid prescribing very high fat days.

**4. Deload Week Preference (scheduled-only vs. auto-triggered)**
- *Why it matters:* Some users resist scheduled deloads even when physiologically needed. Others want auto-triggered deloads based on GYR data. Knowing this preference changes how deload weeks are framed in the app.

**5. Next-Block and Annual Goals**
- *Why it matters:* The OS now plans a 12-month roadmap while detailing the current and next block. After the first recomp or foundation block, what does the user want next? Max strength, hypertrophy, mobility/rehab, sport-specific preparation, skill specialization, or another recomp cycle? Knowing this prevents the first block from ending at a cliff.

**6. Supplement Budget**
- *Why it matters:* Some supplement recommendations (creatine, EAAs, electrolytes) are low-cost; others (peptides, specialized stacks) are not. A budget-aware intake prevents recommending products the user won't buy and creates friction on adoption.

**7. Social / Environment Constraints**
- *Why it matters:* Does the user train alone or with a partner? At home or in a shared space? These affect exercise selection (e.g., no barbell Olympic lifts at 6am in an apartment).

**8. Chronotype and Wake Time**
- *Why it matters:* Early morning training protocols assume the user can wake by 6–7am. A confirmed night owl who must force early training will have consistently lower GYR scores and higher injury risk. Knowing chronotype allows honest framing of the morning routine.

**9. Annual Roadmap Intent**
- *Why it matters:* Recomp may be the first block, but the OS needs to know what the next 6-12 months are building toward. Ask for the desired 12-month outcome and likely next-block priority: strength, hypertrophy, mobility/rehab, endurance, skill specialization, or "decide from tests."

**10. Recovery Bias Preference and Constraints**
- *Why it matters:* Some users need recovery-biased programming by default. Ask whether the user is currently under-recovered, returning from a layoff, dealing with pain, sleeping poorly, traveling often, or trying to push performance. This feeds `recovery-model.md`.

**11. Active Pain Severity and Referral Flags**
- *Why it matters:* "Rehab focus area" is not the same as active pain. Intake should capture pain location, score, what triggers it, what relieves it, and whether there are red flags that require medical referral before programming.

**12. Exercise Media Preference**
- *Why it matters:* Users may want videos, photos, written cues, or all three. This determines how prominent the exercise detail drawer should be and whether external reference links are acceptable while owned media is missing.

**13. Logging Tolerance**
- *Why it matters:* Daily logs and weekly check-ins drive phase advancement. Ask whether the user will log minimum data only, standard data, or detailed metrics including photos/waist measurements.

**14. Next-Block Decision Style**
- *Why it matters:* The system uses objective gates plus coach override. Intake should confirm whether the user wants strict gates, recommendation with override, or guidance only. Default should be recommendation with override.
